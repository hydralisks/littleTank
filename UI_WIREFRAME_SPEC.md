# Little Tank UI 低保真线框说明

## 1. 目标

本文档用于把 `GAME_DESIGN_DOC.md` 中的界面方案进一步细化为可直接开始实现的低保真说明。

MVP 只保留三段式信息结构：

1. 敌方团队框体区
2. 玩家状态区
3. 底部横向技能栏

目标是让玩家通过一条尽量短的视线路径完成：

`观察敌人 -> 判断局势 -> 确认自己状态 -> 释放技能`

## 2. 屏幕布局

推荐基准分辨率：

- `1600 x 900`
- 最低兼容：`1280 x 720`

纵向结构建议：

- 顶部至中上：敌方团队框体区，约 `55%-60%` 高度
- 中下：玩家状态区，约 `12%-15%` 高度
- 底部：技能栏，约 `18%-20%` 高度

页面左右留白不追求真实 MMO UI 密度，MVP 以可读性优先。

## 4. 敌方团队框体区

## 4.1 结构

每个敌方框体由以下部分组成：

- 左侧标记区
- 名称与类型区
- 中央 HP 区
- 目标/威胁区
- 读条区
- 状态图标区

## 4.2 单个框体布局

```text
+--------------------------------------------------------------------------------+
| [骷髅] 骨冠监工         HP███████████░░░ 84%     目标:TANK     重击 1.2s        |
|         状态:[强化][不可晕][减速]                                              |
+--------------------------------------------------------------------------------+
```

## 4.3 视觉层级

敌方框体应首先突出以下信息：

1. 谁在打队友
2. 谁在读高危技能
3. 谁是骷髅头高危单位
4. 当前玩家选中的目标是谁

建议表现方式：

- 当前选中目标：外发光边框
- 攻击队友：整体转红
- 高危读条：读条槽紫色闪动
- 骷髅头单位：左侧固定骷髅图标
- 控制中：框体局部蒙版偏蓝灰

## 4.4 必要字段

MVP 必须显示：

- 名称
- 血条
- 百分比
- 当前目标
- 读条
- 2-4 个状态图标
- 骷髅标记

MVP 可以不显示：

- 详细数值仇恨
- 护甲/抗性
- 复杂图标说明文字

## 5. 玩家状态区

玩家状态区必须位于敌方框体和技能栏之间。

## 5.1 布局建议

分两行即可：

第一行显示资源性信息：

- HP
- 主资源
- 当前目标
- GCD

第二行显示局势性信息：

- 当前减伤 Buff
- 队伍压力
- 当前高危提醒

## 5.2 线框示例

```text
+--------------------------------------------------------------------------------+
| HP 820/1000   怒气 48/100   当前目标:邪焰咏者   GCD 0.3s                        |
| 减伤:盾墙(2.1s)   队伍压力:42/100   危险提示: 烈焰箭即将完成                    |
+--------------------------------------------------------------------------------+
```

## 5.3 重点

这里不是角色面板，而是战斗判断辅助面板。

因此：

- 重要数字要大
- 文本提示要短
- 不要加入无关装饰
- 不要塞入装备、属性、副面板入口

## 6. 横向技能栏

## 6.1 结构

技能栏位于最底部，固定横向 8 格。

每格显示：

- 快捷键
- 技能图标
- 技能名简称
- CD 数字
- 可用状态
- 资源不足状态

## 6.2 线框示例

```text
+--------------------------------------------------------------------------------+
| [1] 嘲讽  3s | [2] 打断 Ready | [3] 控制 6s | [4] 群嘲 18s | [Q] 盾墙 22s      |
| [E] 顺劈 Ready | [R] 爆发 12s | [F] 保命 40s                                    |
+--------------------------------------------------------------------------------+
```

## 6.3 交互状态

技能格至少支持以下状态：

- `Ready`
- `Cooldown`
- `NoResource`
- `LockedByGCD`

表现建议：

- 冷却中：深色蒙版 + 中央倒计时
- 可用：高亮
- 资源不足：边框偏黄/橙
- GCD 锁定：整体轻微压暗

## 7. 交互说明

## 7.1 鼠标交互

- 点击敌方框体：选中目标
- 悬停敌方框体：显示简易提示
- 悬停技能：显示技能说明

## 7.2 键盘交互

- `Tab`：切换到下一个敌方目标
- `Shift+Tab`：切换到上一个敌方目标
- `1/2/3/4/Q/E/R/F`：释放技能

MVP 不需要复杂可自定义按键，但要在代码结构上保留扩展空间。

## 8. 视觉风格建议

MVP 风格不追求炫技，而追求“像 raid frame 工具，但更干净”。

建议：

- 背景色：深灰或冷灰
- 血条：高饱和但有限颜色
- 高危色：红/紫
- 玩家状态区：比背景稍亮一层
- 技能栏：偏沉稳金属或石板风格

不要做：

- 大面积噪点背景
- 复杂奇幻边框
- 过多粒子
- 夸张的动态抖动

## 9. 响应式缩放原则

如果后续要适配较小窗口：

- 优先缩小字体和内边距
- 不要压缩框体数量
- 不要拆分到第二列
- 保持三段式上下结构稳定

## 10. 实现优先级

推荐前端实现顺序：

1. 先做静态整体布局
2. 再做单个敌方框体组件
3. 再做选中态、骷髅标记、读条条
4. 再做玩家状态区
5. 最后做技能栏状态变化和 CD 动画

## 11. 验收点

UI MVP 达到以下条件即可：

- 玩家一眼能区分高危敌人和普通敌人
- 玩家能快速看出谁在攻击队友
- 玩家能在同一条视线上完成目标识别、自身状态确认和技能释放
- 技能 CD 可读性足够，不需要猜测剩余时间
- 骷髅头单位不会被误认为独立 Boss 面板
