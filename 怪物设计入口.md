# 怪物设计入口

这份文档给怪物策划、数值策划和后续接手 agent 使用，说明敌方怪物相关内容应该写在哪些 `.xlsx` 表里，以及当前运行时已经真实读取哪些字段。

## 1. 总入口与责任边界

怪物设计主要编辑：

- `public/designer-data/enemy_data.xlsx`

关卡中把怪物放进战斗时，还会用到：

- `public/designer-data/encounter_balance.xlsx -> 敌人布置`
- `public/designer-data/encounter_balance.xlsx -> 开场状态`

三类表的职责不要混用：

- `enemy_data.xlsx` 负责“怪物目录”：怪物是谁、会哪些技能、技能按什么循环释放、技能会造成什么效果、敌方/玩家/队伍状态是什么。
- `encounter_balance.xlsx -> 敌人布置` 负责“关卡实例”：某一关刷出哪个怪物、放在 5x5 血条格的哪个位置、是否覆盖血量/循环/首领标记/开场读条。
- `player_build.xlsx` 负责“玩家如何应对”：例如战士 T 的打断、控制、盾牌格挡、盾牌反射等。

不要通过重新生成脚本覆盖策划正在编辑的 workbook。除非用户明确要求，不要运行 `npm run generate:designer-data` 或 `node scripts/generateDesignerWorkbooks.mjs`。

## 2. 推荐设计顺序

1. 先确定怪物定位：近战小怪、远程施法怪、刺客/乱仇恨怪、破防重怪、首领。
2. 在 `敌人技能` 里定义技能池，先把所有会被引用的 `skillId` 建好。
3. 在状态表里定义技能会施加的状态：`敌方Buff`、`玩家Debuff`、`队伍Debuff`。
4. 在 `敌人定义` 里建立怪物目录，填写 `skillIdsCsv` 和 `skillCycleCsv`。
5. 到 `encounter_balance.xlsx -> 敌人布置` 里按关卡摆放怪物实例。
6. 如果某一关需要特殊开场读条、特殊血量或特殊技能循环，在 `敌人布置` 里覆盖，不要新建一份几乎相同的怪物目录。

## 4. 怪物定义怎么填

工作表：`敌人定义`

### 关键字段

| 字段名 | 中文含义 | 类型 | 必填 | 填写说明 |
| --- | --- | --- | --- | --- |
| `enemyId` | 怪物目录 ID | string | 是 | 建议用区域或阵营前缀，例如 `harbor_raider`、`highland_commander`；关卡编队通过它引用怪物 |
| `name` | 显示名 | string | 是 | 血条中显示的基础名字 |
| `baseMaxHp` | 基础最大生命值 | number | 是 | 关卡实例可以通过 `encounter_balance.xlsx -> 敌人布置.maxHpOverride` 覆盖 |
| `skillIdsCsv` | 拥有技能 ID 列表 | csv string | 是 | 逗号分隔，必须引用 `敌人技能.skillId` |
| `skillNamesCsv` | 拥有技能名称列表 | csv string | 否 | 仅供策划阅读核对，程序以 `skillIdsCsv` 为准 |
| `skillCycleCsv` | 技能循环列表 | csv string | 是 | 按顺序填写技能 ID，循环到末尾后从头继续；每个技能都必须存在于本行 `skillIdsCsv` |
| `threatLogic` | 仇恨逻辑 | string | 是 | 当前支持 `normal / irregular / bloodlust`；未来占位且没有技能循环的敌人可暂不填 |
| `counteredDurationMs` | 被反制持续时间 | number | 是 | 成功被打断后进入 `countered` 的时长，单位毫秒 |
| `isSkull` | 是否首领标记 | boolean | 否 | 当前没有单独 boss 框架，它只影响显示和部分玩家技能限制 |

### `threatLogic`

- `normal`：标准仇恨。`allyThreat > tankThreat` 时目标为队伍，否则目标为玩家。
- `irregular`：乱仇恨。目标判断与 `normal` 一样按 `tankThreat / allyThreat` 比较；它不会自然增长队伍仇恨、不会衰减玩家仇恨，但每完成一整轮 `skillCycleCsv` 后会清空 `tankThreat / allyThreat`。
- `bloodlust`：嗜血目标。不看 `tankThreat / allyThreat`；优先打玩家和队伍中当前生命百分比较低的一方，适合首领或压血线怪。

当前运行时参数：

| `threatLogic` | 是否自然增长/衰减仇恨 | 目标选择 | 整轮技能循环后是否清仇恨 | 施法目标是否锁定 |
| --- | --- | --- | --- | --- |
| `normal` | 否 | `allyThreat > tankThreat` 打队伍，否则打玩家 | 否 | 是 |
| `irregular` | 否 | 同 `normal` | 是 | 是 |
| `bloodlust` | 否 | 打玩家/队伍中生命百分比较低的一方 | 否 | 是 |

补充规则：

- 基础仇恨来源只来自明确规则：玩家自动攻击、队伍自动/被动伤害、玩家技能伤害公式、玩家技能固定仇恨、开场 `openingTankThreat / openingAllyThreat`，以及明确写入的特殊技能或状态。
- 敌人开始释放技能时会根据当时的仇恨规则选定目标；读条开始后目标锁定，中途 OT 不改变本次技能目标。
- 敌人不在读条中时，UI 目标显示会实时根据当前仇恨规则更新。
- 非 `bloodlust` 的 UI 仇恨状态：`tankThreat < allyThreat` 为丢失；`allyThreat <= tankThreat < allyThreat + 本关 partyAutoDamageMax` 为警告；`tankThreat >= allyThreat + 本关 partyAutoDamageMax` 为稳定。
- `bloodlust` 的 UI 仇恨状态直接看当前目标：目标为玩家时稳定，目标为队伍时丢失。
- `taunted` / `mass-taunt` 状态会强制目标表现为玩家，但不会改变 `threatLogic` 类型本身。

### `skillIdsCsv` 和 `skillCycleCsv` 的关系

- `skillIdsCsv` 是“会哪些技能”。
- `skillCycleCsv` 是“按什么顺序放技能”。
- `skillCycleCsv` 中出现的每个技能都必须在 `skillIdsCsv` 中出现。
- 一个技能可以在 `skillCycleCsv` 中重复出现，用来提高释放频率。

示例：

```text
skillIdsCsv: bone-jab,reckless-rush
skillCycleCsv: bone-jab,reckless-rush,bone-jab
```

含义：这个怪会两个技能，但普通刺击频率更高。

## 5. 怪物技能怎么填

工作表：`敌人技能`

### 技能节奏

| 字段名 | 中文含义 | 类型 | 必填 | 填写说明 |
| --- | --- | --- | --- | --- |
| `skillId` | 技能 ID | string | 是 | 程序内唯一标识 |
| `skillName` | 技能名称 | string | 是 | 施法条显示名称 |
| `targetRuleId` | 技能目标规则 | string | 是 | `threatTarget / tankAndParty / party / otherEnemy / self` |
| `castTimeMs` | 施法时间 | number | 是 | 单位毫秒；越短越难反应，按表值生效，不再有程序内最低 700ms 下限 |
| `channelingMs` | 引导持续时间 | number | 否 | 默认 0；成功完成 `castTimeMs` 后继续引导的时长 |
| `recoveryMs` | 施法后停顿时间 | number | 是 | 只在该技能成功释放完成后开始计算，类似技能后摇，用于控制怪物出手频率 |
| `damageType` | 伤害类型 | string | 否 | 当前支持 `physical / magic`；留空时按物理伤害处理 |
| `playerDamage` | 对玩家伤害 | number | 否 | 仅在命中玩家时生效 |
| `partyDamageOnHit` | 命中玩家时队伍伤害 | number | 否 | 可为 0 |
| `partyDamageOnMiss` | 未命中玩家时队伍伤害 | number | 否 | 常用于 OT 或固定打队伍 |
| `pressureOnHit` | 命中玩家时队伍压力变化 | number | 否 | 可为 0 |
| `pressureOnMiss` | 未命中玩家时队伍压力变化 | number | 否 | 常用于 OT 或固定打队伍 |
| `appliedTargetStatusIdsCsv` | 对目标施加状态 ID 列表 | csv string | 否 | 成功完成 `castTimeMs` 后，对本次锁定目标施加；引用 `敌方Buff / 玩家Debuff / 队伍Debuff` |
| `appliedSelfStatusIdsCsv` | 对自身施加状态 ID 列表 | csv string | 否 | 成功完成 `castTimeMs` 后，对施法敌人自身施加；引导施法的自身状态主要填这里 |
| `castBreakRule` | 施法阻止规则 | string | 是 | `interruptOrControl / controlOnly / unstoppable` |
| `dangerLevel` | 危险等级 | string | 否 | 当前支持 `low / medium / high`；留空按 `low` |

### 目标规则 `targetRuleId`

- `threatTarget`：打当前仇恨目标。坦克拉住时打玩家，仇恨丢失时打队伍。
- `tankAndParty`：伤害/压力仍按当前锁定目标走原有分支；但 `appliedTargetStatusIdsCsv` 中的玩家 Debuff 和队伍 Debuff 会同时生效。
- `party`：固定打队伍，不看仇恨。
- `otherEnemy`：对其他敌人施放，常用于治疗、强化友军。
- `self`：对自己施放，常用于自保或狂暴。
- `mostInjured`：对敌人中当前损失生命值最多的存活目标施放，可能选中自己；损失生命值按 `maxHp - hp` 计算，常用于治疗类技能。

### 伤害与压力分支

`threatTarget` 技能有两个分支：

- 命中玩家时读取：
  - `playerDamage`
  - `partyDamageOnHit`
  - `pressureOnHit`
- 未命中玩家、打到队伍时读取：
  - `partyDamageOnMiss`
  - `pressureOnMiss`

`party` 技能主要读取：

- `partyDamageOnMiss`
- `pressureOnMiss`

`appliedTargetStatusIdsCsv` 会按状态 ID 所在状态表决定落点：

- `threatTarget` 命中玩家时，只施加其中的 `玩家Debuff`。
- `threatTarget` 命中队伍时，只施加其中的 `队伍Debuff`。
- `party` 固定打队伍时，只施加其中的 `队伍Debuff`。
- `tankAndParty` 是例外：不论本次锁定目标是谁，其中的 `玩家Debuff` 和 `队伍Debuff` 都会同时生效。

`otherEnemy / self` 技能通常不填伤害，主要通过 `appliedTargetStatusIdsCsv` 或 `appliedSelfStatusIdsCsv` 施加状态。

### 引导施法 `channelingMs`

- `channelingMs` 留空或填 `0` 时是普通施法。
- `channelingMs > 0` 时，敌人先读 `castTimeMs`；读条成功完成后立即结算伤害和状态，然后进入引导阶段。
- 引导阶段的目标继承本次技能开始施法时锁定的目标；引导期间仇恨变化不会改变这次技能目标。
- UI 中引导阶段施法条按剩余引导时间从右向左倒计时。
- 引导自然结束后，触发施法敌人身上各状态自己的 `onChannelStop` 逻辑，进入 `recoveryMs`，并推进到技能循环下一个技能。
- 引导期间被控制或打断等方式中止时，同样触发施法敌人身上各状态自己的 `onChannelStop` 逻辑；不能施法状态结束或 `counteredDurationMs` 结束后进入技能循环下一个技能，不重读原技能。
- 如果引导效果通过“敌人给自己一个状态”实现，这个状态 ID 填在 `appliedSelfStatusIdsCsv`。只有该状态的 `effectLogicId` 注册了“引导停止时结束”的处理器，才会在引导结束或中止时被移除；不会因为是引导技能施加，或 `effectLogicId` 名字包含 `channel`，就自动清除。

### 伤害类型 `damageType`

- `physical`：物理伤害。会被战士 T 的盾牌格挡等物理减伤影响。
- `magic`：魔法伤害。当前不会被盾牌格挡影响。

### 危险等级 `dangerLevel`

- `high`：不管本次施法目标是玩家坦克还是队伍，敌人血条框都会播放高危提示动画，并产生战斗警告文案。
- `medium`：只有本次施法目标为 `party` 时，敌人血条框会播放高危提示动画；不会产生战斗警告文案。
- `low`：不触发高危提示动画，也不产生战斗警告文案。
- `dangerLevel` 只负责提示强度，不改变伤害、仇恨、读条、目标或打断规则。

与当前战士 T 技能的关键联动：

- `shieldBlock` 只降低 `damageType=physical` 的玩家承伤。
- `shieldReflection` 只免疫并反弹敌方技能对玩家造成的那部分效果和伤害；同一技能的队伍伤害/压力仍然生效。
- `shieldReflection` 的反弹伤害不计算玩家身上的减伤 buff，也不产生仇恨。

### 施法阻止规则 `castBreakRule`

- `interruptOrControl`：可被打断或控制阻止。适合危险但可反制的读条。
- `controlOnly`：只能被控制阻止。适合近战冲撞、部分重击。
- `unstoppable`：不可阻止。适合首领重压技能或教学用硬伤害。

战士 T 当前联动：

- 打断技能只影响 `interruptOrControl`。
- 控制技能可以影响 `interruptOrControl` 和 `controlOnly`。
- 如果玩家技能 `canAffectSkull=false`，则不能控制首领标记怪物。

节奏结算细则：

- `recoveryMs` 不是读条开始后的固定冷却，而是成功释放完成后的后摇。例：`castTimeMs=1500,recoveryMs=1000`，没有干扰时每 2.5 秒成功释放一次。
- 读条中被控制类技能阻止时，不结算该技能的 `recoveryMs`；控制结束后立刻重新从完整 `castTimeMs` 读同一个技能。
- 读条中被打断类技能阻止时，不结算该技能的 `recoveryMs`；敌人等待 `敌人定义.counteredDurationMs` 后，推进到技能循环里的下一个技能。
- `counteredDurationMs` 缺省或为 `-1` 时，回退使用触发打断的玩家技能效果 `durationMs`；仍缺省则为 `0`。

## 6. 施法序列设计指南

### 普通近战怪

建议循环：

```text
bone-jab,reckless-rush,bone-jab
```

特点：

- 轻伤害技能穿插中等压力技能。
- 主要检验坦克是否持续稳住仇恨。

### 远程施法怪

建议循环：

```text
flame-lance,ember-bolt,dark-mend,staff-smash,ember-rush
```

特点：

- 有可打断技能。
- 有固定打队伍技能。
- 有友方强化/治疗技能。
- 迫使玩家在打断、控制、防御之间取舍。

### 乱仇恨怪

建议：

- `threatLogic=irregular`
- 循环短而尖锐，例如 `feint-slash,backstab`

特点：

- 每完成一整轮 `skillCycleCsv` 后清空仇恨，制造不稳定目标。
- 适合测试嘲讽、群嘲、范围控制。

### 破防重怪

建议循环：

```text
guard-breaker,rampage
```

特点：

- 长读条高伤害。
- 破甲或队伍压力明显。
- 适合测试盾牌格挡、无视苦痛、盾牌反射。

### 首领/精英怪

建议：

- `isSkull=true`
- `threatLogic=bloodlust` 或专门设计的高压循环。
- 混合不可阻止技能和可控制技能。

示例：

```text
crusher-slam,ruin-volley,crusher-slam
```

## 7. 状态设计

三张状态表字段一致：

- `敌方Buff`
- `玩家Debuff`
- `队伍Debuff`
- `图标资源映射`

### 状态表怎么选

- 作用在敌人身上：填 `敌方Buff`。例如强化、自疗、狂暴、被反制。
- 作用在玩家身上：填 `玩家Debuff`。例如破甲、昏迷、沉默。
- 作用在队伍身上：填 `队伍Debuff`。例如压制、阵型冲散、齐射混乱。

### 状态字段

| 字段名 | 中文含义 | 类型 | 必填 | 填写说明 |
| --- | --- | --- | --- | --- |
| `statusId` | 状态 ID | string | 是 | 程序内唯一标识 |
| `statusName` | 状态名称 | string | 是 | 图标悬浮说明与文案用名 |
| `iconId` | 图标资源 ID | string | 是 | 引用 `图标资源映射.iconId`；运行时状态图标优先使用该字段 |
| `durationMs` | 持续时间 | number | 是 | 单位毫秒；`-1` 表示永久持续到战斗结束，除非被明确移除 |
| `isDispellable` | 是否可驱散 | boolean | 是 | 当前先作为配置记录 |
| `description` | 状态效果说明 | string | 是 | 面向策划与后续 UI 说明 |
| `effectLogicId` | 状态效果逻辑接口 ID | string | 是 | 程序用受控 ID 连接逻辑 |

### `iconId`

- 三张状态表都必须填写 `iconId`。
- `iconId` 引用 `enemy_data.xlsx -> 图标资源映射.iconId`。
- 状态实际显示图标按 `iconId -> assetKey -> public/status-icons/{assetKey}.svg` 解析。
- 如果一个状态想复用另一个图标，可以在 `图标资源映射.assetKey` 中填写相同资源键；例如 `suppression` 当前复用 `heat-haze`。
### `effectLogicId`

`effectLogicId` 是程序受控接口 ID，不是脚本内容。

当前常用：

- `none`：只展示或逻辑待实现。
- `enemy_heal_small`：敌方小额自疗示例。
- `channel_self_until_end`：敌方自身状态在来源引导技能自然结束、被控制中止或被打断中止时移除；用于描述中写明“在 xxxx 引导停止时结束”的状态。
- `wind_strike!_status`：敌方自身引导状态；引导期间每 0.5 秒对本次锁定目标造成 5 点伤害，并在来源引导技能停止时移除。

如果策划需要新状态产生真实逻辑，先在 `description` 写清楚效果，再让后续 agent 在代码注册表里接入新的 `effectLogicId`。不要把公式或脚本直接写进 Excel。

### 图标资源映射字段

工作表：`图标资源映射`

| 字段名 | 中文含义 | 类型 | 必填 | 填写说明 |
| --- | --- | --- | --- | --- |
| `iconId` | 图标 ID | string | 是 | 供状态表、关卡图例、词缀/规则引用 |
| `iconName` | 图标名称 | string | 是 | 供策划阅读 |
| `assetKey` | 资源键 | string | 是 | 对应 `public/status-icons/{assetKey}.svg` 的文件名主体 |
| `iconType` | 图标类型 | string | 是 | 当前敌方数据中统一填写 `status` |
| `enabled` | 是否启用 | boolean | 否 | 默认 `true` |

## 8. 与关卡设计入口的衔接

怪物目录建好后，到 `encounter_balance.xlsx -> 敌人布置` 使用：

- `enemyId`：引用 `enemy_data.xlsx -> 敌人定义.enemyId`。
- `spawnId`：关卡内唯一实例 ID。
- `row / col`：5x5 血条格位置，范围 1 到 5。
- `hpOverride / maxHpOverride`：只改这一关的血量。
- `openingCastSkillNum`：设置开场从技能循环中的第几个技能开始，1 表示第一个技能。
- `openingRecoveryRemainingMs`：设置开场发呆时间；时间结束后才会按 `openingCastSkillNum` 从头完整读条。
- `openingTankThreat / openingAllyThreat`：设置开场玩家/队伍仇恨。

规则：能作为怪物通用模板复用的内容，写在 `enemy_data.xlsx`；只属于某一关、某一个实例的内容，写在 `encounter_balance.xlsx`。`skillCycleCsv / threatLogic / isSkull` 现在都以 `enemy_data.xlsx -> 敌人定义` 为准，不再由 `encounter_balance.xlsx -> 敌人布置` 覆盖。

## 9. 填表检查清单

提交给后续 agent 或进入测试前，至少检查：

- `敌人定义.enemyId` 没有重复。
- `敌人定义.skillIdsCsv` 中的每个 ID 都存在于 `敌人技能.skillId`。
- `敌人定义.skillCycleCsv` 中的每个 ID 都存在于本行 `skillIdsCsv`。
- `敌人技能.appliedTargetStatusIdsCsv / appliedSelfStatusIdsCsv` 中的每个 ID 都存在于三张状态表之一。
- `targetRuleId`、`damageType`、`castBreakRule`、`dangerLevel` 使用合法值。
- 首领怪如果不希望被普通控制影响，要确认玩家技能侧 `canAffectSkull` 设计。
- 需要检验盾牌格挡的技能应填 `damageType=physical`；需要绕过盾牌格挡的技能应填 `magic`。
- 固定打队伍的技能不要把主效果写在 `playerDamage`。
- 开场读条、特殊血量、特殊摆位写到关卡表，不要复制一份新怪物。
